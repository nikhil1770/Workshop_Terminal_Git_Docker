💡 Tip: Your prompt shows where you are — nikhil@MacBook Desktop % means you're inside the Desktop folder. Always know where you are before running commands.
Navigation
pwd
Print Working Directory — shows your current location
ls
List — shows all files and folders here
ls -la
List all — shows hidden files too (ones starting with .)
cd foldername
Change Directory — go into a folder
cd ..
Go up — move one folder back
cd ~
Go home — jump to your home folder from anywhere
cd ~/Desktop
Go to Desktop — shortcut you'll use a lot
Files & Folders
mkdir foldername
Make Directory — creates a new folder
touch filename.txt
Touch — creates a new empty file
cat filename.txt
Cat — prints the contents of a file
cp file1 file2
Copy — copies a file
mv file1 file2
Move / Rename — moves or renames a file
rm filename
Remove — deletes a file permanently (no trash!)
rm -rf foldername
Remove folder — deletes a folder and everything inside ⚠️
Useful Tricks
clear
Clear screen — cleans up the terminal (or press ⌘ + K)
echo "hello"
Echo — prints text to the terminal
↑ arrow key
History — scroll through previous commands
Tab key
Autocomplete — press Tab to finish folder/file names
Ctrl + C
Cancel — stop any running command

💡 Golden workflow: Make changes → git add . → git commit -m "message" → git push. That's 90% of what you'll do with Git every day.
Setup
git --version
Check Git — confirms Git is installed
git config --global user.name "name"
Set name — tells Git who you are
git config --global user.email "email"
Set email — use your GitHub email
git config --list
Check config — see all your Git settings
Core Workflow
git init
Initialise — turns any folder into a Git repo
git status
Status — shows what's changed. Run this constantly!
git add filename
Stage a file — prepares one file for the next commit
git add .
Stage everything — prepares all changed files
git commit -m "message"
Commit — saves a snapshot with a description
git log --oneline
History — see all your commits in a clean list
git diff
Diff — shows exactly what lines changed
Branches
git branch
List branches — shows all branches (* = current)
git switch -c branchname
Create & switch — makes a new branch and moves to it
git switch branchname
Switch branch — move to an existing branch
git merge branchname
Merge — brings another branch's changes in
git branch -d branchname
Delete branch — removes a branch after merging
GitHub (Remote)
git clone url
Clone — downloads a repo from GitHub to your Mac
git remote -v
Check remote — shows which GitHub repo you're connected to
git push
Push — uploads your commits to GitHub
git pull
Pull — downloads latest changes from GitHub
git fetch
Fetch — checks for changes without applying them
Undoing Mistakes
git restore filename
Discard changes — undo edits to a file (cannot be undone!)
git restore --staged filename
Unstage — remove a file from staging area
git commit --amend -m "msg"
Fix last commit — change the message of your last commit
git reset --soft HEAD~1
Undo last commit — keeps your changes, removes the commit
git stash
Stash — temporarily shelves changes so you can switch branches
git stash pop
Restore stash — brings back your shelved changes
// example — full workflow from init to push
git init
touch app.py
git add .
git commit -m "first commit"
[main c65a48d] first commit
git push -u origin main
Branch 'main' set up to track 'origin/main'
💡 Remember: Docker Desktop must be open and running before any docker command will work. Look for the 🐳 whale icon in your menu bar.
Setup & Check
docker --version
Check Docker — confirms Docker is installed
docker ps
Running containers — lists currently running containers. If this works, Docker engine is ON ✅
docker ps -a
All containers — shows running AND stopped containers
docker images
List images — shows all downloaded images on your Mac
Running Containers
docker run imagename
Run — downloads image (if needed) and starts a container
docker run hello-world
Test run — Docker's built-in test. Use this to verify everything works
docker run -it imagename bash
Interactive mode — opens a terminal inside the container. Type exit to leave
docker run -d imagename
Detached mode — runs container in background
docker run -p 8080:80 imagename
Port mapping — connects container port 80 to your Mac's port 8080
Managing Containers
docker stop containerid
Stop — gracefully stops a running container
docker start containerid
Start — restarts a stopped container
docker rm containerid
Remove container — deletes a stopped container
docker logs containerid
Logs — see what a container has been printing
docker exec -it containerid bash
Enter running container — open a shell inside a container that's already running
Managing Images
docker pull imagename
Pull — downloads an image from Docker Hub without running it
docker rmi imagename
Remove image — deletes a downloaded image
docker build -t name .
Build image — creates an image from a Dockerfile in current folder
Cleanup
docker system prune
Prune — removes all stopped containers, unused images, frees up space
docker stop $(docker ps -q)
Stop all — stops every running container at once

docker pull ubuntu
latest: Pulling from library/ubuntu
docker run -it ubuntu bash

root@abc123:/# exit
