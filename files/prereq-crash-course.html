Open Terminal: press ⌘ + Space, type Terminal, hit Enter. Every command below goes into Terminal.

Navigate folders

2
Create and remove
# Create a new folder mkdir my-project # Create an empty file touch hello.txt # Print text to terminal echo "Hello World" # Clear the screen clear

2 — Install Git & Configure It
~30 min · One-time setup

Git is a tool that tracks changes in your code — like a detailed undo history for entire projects. Install it first, then tell it who you are.

1
Install Homebrew (macOS's package manager — you'll use this forever)
Paste this into Terminal and press Enter. It will take a few minutes:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
When it finishes, follow any instructions it prints about adding Homebrew to your PATH.
2
Install Git via Homebrew
brew install git
3
Verify Git is installed
git --version git version 2.45.x
You should see a version number. If you do, Git is installed ✓
4
Tell Git your name and email — use the same email you'll use for GitHub:
git config --global user.name "Your Name" git config --global user.email "you@email.com" # Confirm it saved git config --list
✔ You're ready when git --version shows a version number.
📦
3 — Learn Core Git Workflow
~1.5 hrs · The 6 commands you must know

Git works by taking snapshots (called commits) of your project over time. Here is the complete beginner workflow:

1
Create a project and initialise Git
cd ~/Desktop mkdir my-first-repo cd my-first-repo git init Initialized empty Git repository in .../my-first-repo/.git/
git init turns any folder into a Git-tracked project.
2
Create a file and check Git's status
touch README.md echo "# My First Project" > README.md git status Untracked files: README.md
git status shows what's changed. "Untracked" = Git sees it but isn't tracking it yet.
3
Stage the file (tell Git "I want to include this in the next snapshot")
git add README.md # Or add everything at once: git add .
4
Commit the snapshot (save it permanently with a message)
git commit -m "Add README file" [main (root-commit)] Add README file
The -m flag lets you write a message describing what you did.
5
View your history
git log --oneline a3f9c12 Add README file
Each line is one commit. The random letters are the commit's unique ID.
The Golden Workflow to memorise:
Make changes → git add . → git commit -m "message" → repeat.
That's 90% of what you'll do with Git.
✔ You're ready when you can init a repo, add files, and commit — all from memory.
🐙
4 — Create a GitHub Account & Push Your Repo
~1 hr · Connect local Git to the internet

GitHub is a website that stores your Git repositories online. Think of Git as your local save system and GitHub as the cloud backup.

1
Create your account
Go to github.com → Sign up → choose a username and use the same email you gave Git. Verify your email.
2
Create a new repository on GitHub
Click the + icon → "New repository" → name it my-first-repo → set it to Public → do NOT add a README (you already have one) → click "Create repository".
3
Connect your local repo to GitHub and push
GitHub will show you commands after creation. Use these in Terminal (replace YOUR-USERNAME):
git remote add origin https://github.com/YOUR-USERNAME/my-first-repo.git git branch -M main git push -u origin main
It will ask for your GitHub username and password. For password, use a Personal Access Token (see next step).
4
Create a Personal Access Token (PAT) — GitHub no longer accepts your real password
GitHub → Settings → Developer Settings → Personal Access Tokens → Tokens (classic) → Generate new token → check repo scope → copy the token. Save it somewhere — you only see it once. Use this as your password when pushing.
5
Refresh github.com/YOUR-USERNAME/my-first-repo — you should see your README.md file there. You just pushed your first repo! 🎉
✔ You're ready when your repository appears on GitHub with your README file.

5 — What Is Docker? (Concept)
~20 min · Understand before you install

The problem Docker solves: "It works on my machine" — code that runs fine for one person but breaks for another because of different OS, different software versions, different settings.

Docker's solution: A container — a self-contained package that includes your app AND everything it needs to run (the right OS version, the right libraries, the right settings). You ship the container, not just the code.

Analogy: Think of a Docker container like a lunchbox. It has the food, utensils, and napkins inside — everything needed to eat, regardless of whose kitchen you're in. Docker Desktop is the app that lets you create and open these lunchboxes.

Key terms you'll hear at the workshop:

  • Image — the blueprint/recipe for a container
  • Container — a running instance of an image (like running a program from an .exe)
  • Docker Hub — a public library of pre-made images (like an app store)
  • Docker Engine — the background service that actually runs containers (must be ON)
⚙️
6 — Install Docker Desktop
~30 min · Installation + verification
1
Download Docker Desktop
Go to docker.com/products/docker-desktop → click "Download for Mac".
⚠️ Choose the right chip: Apple Silicon (M1/M2/M3/M4) or Intel — check by going to 🍎 → "About This Mac".
2
Install it
Open the downloaded .dmg file → drag Docker to Applications → open Docker from Applications → accept the service agreement → wait for Docker to fully start (the whale icon in your menu bar should stop animating).
3
Verify Docker is running in Terminal:
docker --version Docker version 26.x.x docker ps CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
If docker ps returns a table (even empty), the Docker Engine is running. ✓
Important for Friday: Docker Desktop must be open and running when you arrive at the workshop. Add it to your login items: Docker menu bar icon → Preferences → General → check "Start Docker Desktop when you log in".
✔ You're ready when docker ps runs without any errors.
🚀
7 — Run Your First Container
~30 min · Hands-on practice

This is just practice to make sure everything works and so Docker doesn't feel alien on Friday.

1
Run the hello-world container (Docker's built-in test):
docker run hello-world Hello from Docker! This message shows that your installation appears to be working correctly.
Docker pulled the image from Docker Hub and ran it. That's containers in action.
2
Run an Ubuntu Linux shell inside a container (mind-bending, but try it):
docker run -it ubuntu bash root@abc123:/#
You're now inside a Linux container! Type exit to get back out.
3
Useful Docker commands to know:
# List running containers docker ps # List all containers (including stopped) docker ps -a # List downloaded images docker images # Stop a running container docker stop <container-id>
✔ You're ready when you can run docker run hello-world and see the success message.
